# 不動産買取再販 計算・稟議・事業計画書アプリ

> **運用方針（重要）**: 本アプリは **サーバー不要の静的サイト**として GitHub Pages で**無料・商用OK**で運用します。公開手順は **[DEPLOY.md](./DEPLOY.md)** を参照。
> AI事業計画書は **API課金なし**で、ブラウザ版 Claude（claude.ai）に「プロンプトをコピーして貼り付け」る手動方式です（APIキー不要）。

中小不動産会社の買取再販事業向けに、**①物件ごとの収支計算 ②社内稟議書 ③事業計画書（銀行提出用／社内方針確認用）** を、スマホで一気通貫で作成できる Web アプリ（PWA）です。スプレッドシート運用をやめ、端末内（IndexedDB）で完結します。

- **2社共用**：「ゆかり株式会社」「ViVi不動産株式会社」を案件ごとに切替。会社名は稟議書・計算書・全PDFに反映。
- **モバイルファースト**：カード型UI・ボトムナビ・数値キーパッド最適化・ダーク/ライト対応。
- **AI事業計画書（手動・無料）**：計算結果入りプロンプトをコピー → claude.ai に貼付 → 生成文を本文へ貼り戻し（数値は計算結果をそのまま使用）。
- **PDF出力（4種）**：計算書／稟議書／銀行提出用／社内用。日本語フォント（Noto Sans JP）埋め込みで文字化けなし。

## 技術スタック
- Next.js 14 (App Router) + TypeScript + React 18
- Tailwind CSS 3（自作の軽量 shadcn 風 UI）
- 状態管理：Zustand / 永続化：IndexedDB（`idb`）
- グラフ：recharts / PDF：`@react-pdf/renderer`（Noto Sans JP 埋め込み）
- AI：`@anthropic-ai/sdk`（サーバー側 API Route。キーは環境変数）
- PWA：manifest + Service Worker（オフラインでも計算可能）

## ⚠️ 重要：実行場所について（Google Drive 不可）

このプロジェクトは Google ドライブ（`G:\マイドライブ\...`）上に置かれていますが、**Google ドライブの仮想ファイルシステムは `node_modules` を正しく扱えず、`npm install` が `EPERM`/`EBADF` で失敗します。**

そのため **開発・実行はローカルディスク（例：`C:\dev\kaitori-saihan-app`）で行ってください。** ソース一式は Google ドライブのプロジェクトフォルダにも同期してありますが、`node_modules` はローカルにのみ作成します。

### ローカルへのセットアップ手順（PowerShell）

```powershell
# 1) ローカル作業フォルダへソースをコピー（node_modules/.next は除外）
$src = "G:\マイドライブ\claude-work\不動産買取再販 計算・稟議・事業計画書アプリ"
$dst = "C:\dev\kaitori-saihan-app"
robocopy $src $dst /E /XD node_modules .next /XF "【買取後案件】.lnk"

# 2) ローカルで依存インストール
cd $dst
npm install

# 3) APIキーを設定（AI生成を使う場合）
Copy-Item .env.local.example .env.local
#   .env.local を開き ANTHROPIC_API_KEY を貼り付け

# 4) 開発サーバー起動
npm run dev          # http://localhost:3000
```

本番用：

```powershell
npm run build
npm run start        # http://localhost:3000
```

スマホで使う場合は同一LAN内で `npm run dev -- -H 0.0.0.0` を実行し、`http://<PCのIP>:3000` にアクセス → ブラウザの「ホーム画面に追加」で PWA としてインストールできます。

## 使い方
1. **マイ案件**（`/`）：「新規案件」または「サンプル読み込み」（4タイプの実例値を投入）。会社で絞り込み可。
2. **案件編集**（`/projects/[id]`）：タブで切替。
   - **計算書**：物件タイプを選び費目を入力 → 取得原価・粗利・営業利益などをリアルタイム自動計算。仲介手数料・不動産取得税・固都税精算金は「自動計算」ボタンあり。基準（粗利250万 or 粗利率30%）達成を色バッジで表示。
   - **稟議書**：物件情報・スペック・査定・**27項目チェックリスト**・**9ステップ日程**・再販試算サマリー（計算書から自動連携）・承認欄。
   - **シミュレーション**：複数シナリオ（強気/標準/弱気、±10%プリセット）を横並び比較＋グラフ。損益分岐販売価格・目標利益率価格を算出。
   - **事業計画書**：銀行用／社内用を AI 生成 → 本文を手動編集 → 4種 PDF をダウンロード。
3. **設定**（`/settings`）：会社マスタ（初期 2社登録済み・追加/編集/削除・ロゴ）、デフォルト会社、テーマ。

データは入力のたびに自動保存（手動保存ボタンもあり）。再訪時は続きから編集できます。

## 計算ロジック（単位：万円）
```
売上原価   = 取得原価 + 経費
粗利益     = 販売価格 − 売上原価
粗利率     = 粗利益 ÷ 販売価格
営業利益   = 粗利益 − 販売経費
営業利益率 = 営業利益 ÷ 販売価格
補助: 仲介手数料 = 価格×3% + 6万円 / 取得税 = 土地評価額×1.5% + 建物評価額×3% / 固都税精算 = 年額×日数/365
```
ユニットテストでサンプル4タイプの期待値（粗利益・営業利益・各利益率）と一致することを検証しています。

```powershell
npm test
```

## 過去案件フォルダ【買取後案件】から参考にした点
実在の過去案件フォルダ（Google Drive 共有のショートカット）を確認し、**構造・形式のみ**を反映しました。**個人情報（売主氏名・連絡先・実住所等）はコード／サンプルに一切転記していません。**

- **事業計画書の章立て**：実運用の「買取再販事業計画書(.docx)」の構成（物件概要／行動予定日／取得経費・原価／販売経費／予想利益／金融機関／必要時期）を、PDF と AI 生成プロンプトの骨子に採用。
- **費目の表記スタイル**：①〜⑩の番号付き・円単位・「自己資金にて支払い」「売買代金より支払い」等の注記の言い回しを参考に、事業計画書の体裁へ反映。
- **2社の役割分担**：ゆかり＝買取再販の主体、ViVi不動産＝仲介（仲介手数料を受領）という実態を、会社マスタの「役割」初期値・PDF署名に反映。
- **連結粗利の概念**：ゆかりの営業利益＋ViViの仲介手数料粗利という考え方を把握（任意活用）。
- **物件概要の粒度**：土地/建物の面積（㎡＋坪）・地目・権利・構造・床面積・建築時期・用途地域・建蔽率/容積率・接道といった項目構成を、稟議書スペック欄の参考に。
- **資金スケジュール**：手付金（契約時）／決済（取得経費）の二段で必要時期を示す実務を、稟議書スケジュール・事業計画書「必要時期」へ反映。
- **金額レンジ感**：サンプル案件の初期値を実際の規模感（戸建・土地・建売・区分）に寄せて設定。

## 備考・制限
- データはブラウザの IndexedDB に保存されます（端末・ブラウザごと）。任意のバックアップエクスポートは `lib/db.ts` の `exportAll()` を利用可能（主たる保存手段ではありません）。
- 損益分岐価格は「仲介手数料＝価格×3%＋6万円、その他販売経費は固定」とみなした近似値です。
- AI は文章のみを生成し、数値は計算結果をそのまま埋め込むため、数値の捏造は起きません。
